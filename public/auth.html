Authentication

The Staark API uses API keys to authenticate requests. All API calls must be made over HTTPS — calls over plain HTTP will fail.

How it works

Every request to the Staark API must include your API key in the Authorization header using the Bearer scheme. Keys are scoped to a workspace and carry the permissions of the user who generated them.

API keys are prefixed with sk_live_staark_ for production keys. Keep them secure — they provide full access to your workspace.

Making authenticated requests

1
Generate an API key
Go to /v1/get-key, choose your plan, and generate a key. It will only be shown once.
2
Store it as an environment variable
Never hard-code your key in source code. Use environment variables or a secrets manager.
3
Include it in the Authorization header
Pass the key as a Bearer token on every request.
HTTP Header
Authorization: Bearer sk_live_staark_••••••••••••••••
cURL
curl https://api.staark-app.cloud/v1/projects \
  -H "Authorization: Bearer sk_live_staark_••••••••" \
  -H "Content-Type: application/json"
Node.js
const res = await fetch('https://api.staark-app.cloud/v1/projects', {
  headers: {
    'Authorization': `Bearer ${process.env.STAARK_API_KEY}`,
    'Content-Type': 'application/json',
  }
});

Security best practices

⚠
Your API key grants full access to your workspace. Treat it like a password — never expose it in client-side code, public repos, or logs.
✓
Use environment variables
Store keys in .env files and load them via process.env. Add .env to your .gitignore.
✓
Rotate keys regularly
Generate a new key periodically and revoke old ones from the dashboard.
✓
Never expose in frontend code
API calls with your key should always go through your backend, never from the browser directly.
✓
Use HTTPS only
All requests must use HTTPS. HTTP requests will be rejected with a 403 error.

Authentication errors

If your request is not authenticated correctly, you will receive a 401 Unauthorized response:

JSON — 401 Response
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or missing API key",
    "docs": "https://api.staark-app.cloud/authentication"
  }
}
