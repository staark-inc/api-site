Release Notes

Changelog

All notable changes to the Staark API are documented here. We follow semantic versioning.

v1.5.0
Mar 31, 2026
SSH Support, New Design & Auto-Deploy
Major release introducing SSH connection management, a fully redesigned interface with a new color system, and a CI/CD pipeline with automated deployment via GitHub webhooks.
Feature Performance Fix
  • feat GET /v1/ssh/connections — list SSH connections
  • feat PUT /v1/ssh/connections/:id — create or update a connection
  • feat DELETE /v1/ssh/connections/:id — remove a connection
  • feat POST /v1/ssh/connections/sync — bulk sync connections
  • feat New SSH panel in dashboard with full CRUD interface
  • feat Auto-deploy via GitHub registry_package webhook on every push to main
  • feat CI/CD pipeline with GitHub Actions — build & push to GHCR automatically
  • feat New design system — custom color palette across all public pages
  • fix Corrected endpoint count from 32 to 29 in documentation
v1.4.2
Mar 04, 2026
Webhook improvements & bug fixes
Minor release with expanded webhook event coverage and stability fixes across the tasks endpoint.
Feature Fix
  • feat Added task.deleted webhook event
  • feat Webhook payloads now include actor_id field
  • fix Fixed pagination on GET /tasks returning duplicate items
  • fix due_date now correctly accepts ISO 8601 format
v1.4.1
Feb 18, 2026
Performance & security patch
Response times on large project queries improved by ~40%. Rate limiting headers are now included in all responses.
Performance Security
  • perf Optimized GET /projects query — up to 40% faster
  • feat Added X-RateLimit-Remaining and X-RateLimit-Reset headers
  • fix API keys are now compared using constant-time comparison
v1.4.0
Jan 30, 2026
Webhooks — General Availability
Webhooks are now out of beta and available on all plans. Pro plan gets higher delivery guarantees and retry logic.
Feature
  • feat POST /webhooks — register endpoint for real-time events
  • feat HMAC-SHA256 signature verification via X-Staark-Signature
  • feat Automatic retry with exponential backoff (Pro only)
  • feat Supported events: task.created, task.updated, project.archived, member.invited
v1.3.0
Dec 12, 2025
Breaking: pagination & auth changes
Standardized pagination across all list endpoints. Authentication now requires Bearer prefix — plain token no longer accepted.
Breaking Feature
  • break Auth header must now use Bearer prefix explicitly
  • break Pagination response shape changed — total moved inside meta object
  • feat Added cursor-based pagination on GET /tasks
  • fix Removed deprecated /v0 endpoints
