Error Handling

The Staark API uses conventional HTTP response codes to indicate the success or failure of a request. Errors include a machine-readable code and a human-readable message.

Error response format

All errors return a consistent JSON structure with an error object:

JSON
{
  "error": {
    "code": "VALIDATION_ERROR",        // machine-readable code
    "message": "'title' is required",    // human-readable description
    "field": "title",                  // field that caused the error (if applicable)
    "docs": "https://api.staark-app.cloud/errors"
  }
}

HTTP status codes

2xx
Success
200 OK for reads, 201 Created for successful creates. The request was processed successfully.
400
Bad Request
The request was malformed — missing required fields, invalid types, or failed validation. Check the field property in the error response.
401
Unauthorized
Missing or invalid API key. Make sure your Authorization: Bearer header is present and the key is valid.
403
Forbidden
Your API key does not have permission to perform this action. Check the key's workspace scope.
404
Not Found
The requested resource does not exist or was deleted. Double-check the ID in the URL path.
429
Too Many Requests
Rate limit exceeded. Check the Retry-After header and implement exponential backoff. See Rate Limits.
5xx
Server Error
Something went wrong on our end. These are rare — check the Status page for ongoing incidents and retry with backoff.

Error codes reference

CodeHTTPDescription
UNAUTHORIZED 401 API key is missing or invalid
FORBIDDEN 403 Key does not have access to this resource
NOT_FOUND 404 Resource with given ID does not exist
VALIDATION_ERROR 400 Request body failed schema validation
MISSING_FIELD 400 A required field was not provided
INVALID_TYPE 400 Field value is the wrong type
RATE_LIMIT_EXCEEDED 429 Too many requests in the current window
CONFLICT 409 Resource already exists or action conflicts with current state
INTERNAL_ERROR 500 Unexpected server error — retry with backoff
SERVICE_UNAVAILABLE 503 API is temporarily unavailable

Handling errors in code

Node.js
const res = await fetch('https://api.staark-app.cloud/v1/tasks', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${apiKey}` },
  body: JSON.stringify({ title: 'New task' })
});

if (!res.ok) {
  const { error } = await res.json();

  switch (error.code) {
    case 'UNAUTHORIZED':
      throw new Error('Invalid API key');
    case 'RATE_LIMIT_EXCEEDED':
      await sleep(error.retry_after * 1000);
      break;
    case 'VALIDATION_ERROR':
      console.error(`Field error: ${error.field} — ${error.message}`);
      break;
    default:
      throw new Error(error.message);
  }
}
ℹ
For 5xx errors, always retry with exponential backoff — start with 1s, then 2s, 4s, up to a maximum of 60s. Most transient server errors resolve within a few seconds.
